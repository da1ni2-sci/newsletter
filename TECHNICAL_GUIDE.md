# 鍛碼匠技術週報 (Fordige Tech Newsletter) 核心開發技術文檔

本文件詳盡紀錄「鍛碼匠技術週報」生成系統的開發思想、架構設計、核心算法及實作細節。

---

## 1. 開發思想與品牌願景 (Vision & Philosophy)

### 1.1 品牌核心：鍛造 (Forging)
系統設計不追求單純的「AI 翻譯」或「自動摘要」，而是強調 **「手動鍛造」** 的精神。這意味著：
- **去模組化**：拒絕機器人式的固定格式（如「首先、接著、最後」）。
- **轉譯而非翻譯**：將硬核的技術「天書」轉化為「早餐配咖啡的八卦」。
- **三明治包裝法**：先給比喻（麵包），再給術語（肉排），最後給結果（麵包），降低認知阻力。

### 1.2 受眾精準導向 (Audience Centric)
每一期週報必須維持三類受眾的內容配比：
- **🧑‍💻 技術開發者**：深度的架構拆解與硬核 Repo。
- **💼 商業決策者**：技術的 ROI、落地成本與市場趨勢。
- **🧐 科技愛好者**：新鮮的技術趣聞與未來生活想像。

## 2. 系統架構 (System Architecture)

系統採用 **Agent Swarm (代理人群體) 模式**，透過多個具備不同人設的 Agent 協作完成從資料抓取到最終潤飾的流程。

### 2.1 核心技術棧
- **UI 框架**: Streamlit (管理後台與工作流監控)。
- **向量資料庫**: Qdrant (本地儲存，負責長期記憶與歷史查重)。
- **嵌入模型**: Ollama `qwen3-embedding` (4096 維高解析度向量)。
- **大型語言模型**: 支援 Ollama (本地)、DeepSeek API 與 OpenAI API 的混合調度。
- **搜尋引擎**: SerpAPI (整合 Google Search 進行深度研究)。
- **爬蟲技術**: Playwright (Stealth 模式) 結合自定義 Parser。

## 3. 核心流程詳解 (Component Deep-Dive)

### 🔍 階段 1：多源資料抓取 (Ingestion)
- **多維度監控**：涵蓋 Arxiv (學術)、GitHub Trending (工程)、Reddit/HN (社群討論) 及各大 AI 實驗室 Blog。
- **反爬蟲對策**：實作 `LocalPlaywrightStrategy`，利用無頭瀏覽器模擬真人行為，解決 Reddit 等平台 403 封鎖問題。
- **內容富化**：抓取時自動提取 README、論文摘要或網頁全文，為後續聚類提供充足語義資訊。

### 🧬 階段 2：語義增強與聚類 (Aggregation)
- **意圖標籤 (Intent Tagging)**：在物理聚類前，先由 Agent 為文章產生 3-5 個技術標籤，打破來源平台的語義隔閡。
- **智慧閾值 (Auto-Tune)**：系統自動掃描 0.5 到 0.8 的距離閾值，由 LLM 分析聚類統計數據（孤立點、巨型類簇），動態挑選最優參數。
- **層次聚類**：採用 `AgglomerativeClustering` (Ward Linkage)，確保技術主題的物理分群具備高度凝聚力。

### 🤖 階段 3：AI 總編選題與查重 (Editor Selection)
- **歷史記憶檢索**：選題前自動檢索 Qdrant 中的 `finalized_newsletters` 集合。
- **防重複機制**：若候選主題與過去 12 個月的內容高度相似且無重大更新，系統將自動過濾或標註為 `EXTENDED UPDATE`。
- **受眾平衡算法**：強制 Agent 在挑選 Top 5 時符合 [2:2:1] 的受眾配比。

### 🔍 階段 4：深度研究 (Deep Research)
- **主動查證**：針對選中主題，由 `ResearchAgent` 生成針對性的 Google 搜尋 Query。
- **多輪檢索**：整合 SerpAPI 進行真實搜尋，並利用爬蟲抓取搜尋結果，產出「研究筆記」。
- **斷點續傳**：系統優先讀取 UI 上的手動研究報告，若為空才啟動自動搜尋，尊重人工干預。

### ✨ 階段 5：終極鍛造 (Chief Editor Refinement)
- **Freestyle 寫作指令**：內化 `Paradox -> Gap -> Reveal -> Insight` 邏輯，禁止使用重複的起手式。
- **全局流動性 (Flow)**：自動分析 5 篇文章的關聯，重新編排順序並撰寫幽默的轉場文字。
- **總編導讀**：根據當期內容產出具備溫度的開場白。
- **術語書籤系統**：自動標註術語連結 `[術語](#term-術語)` 並在文末產出具備 HTML Anchor 的「劃重點」區塊。

## 4. 關鍵技術細節 (Key Technical Implementation)

### 4.1 連結與開啟方式 (UX Optimization)
- **新分頁開啟**：所有外部連結（正文參考、延伸閱讀、頁尾）強制使用 HTML `<a target="_blank">`，避免讀者跳出率過高。
- **內部分頁跳轉**：利用 HTML `<a name="ID">` 實作術語與重點區塊的雙向跳轉。

### 4.2 期刊號索引系統
- **序列化標註**：格式採用 `YYYY-MM-00x`。
- **存檔邏輯**：點擊完稿後，系統將 Markdown 內容按 ## 標題切分為 Chunks，進行向量化後存入 Qdrant， Payload 包含 `issue_number` 以利未來回溯。

### 4.3 幻覺過濾機制 (Over-Generation Filter)
- **Regex 強制切除**：在 Agent 產出文章時，常會因幻覺自行在每篇文末加上「劃重點」。系統在串接最終文本前，會透過正則表達式強制移除這些局部區塊，確保「劃重點」只出現在全文最末端。

### 4.4 HTML 郵件模板
- **Email Safe CSS**：內建一套適用於各大 Mail 客戶端的 CSS 樣式表，包含寬度限制 (800px)、品牌色系 (#1b8f7a) 與美觀的程式碼區塊排版。
